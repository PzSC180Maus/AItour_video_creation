# AItour Video Creation

忆景创影：一个面向文旅场景的 AI 视频生成小程序原型。

本文档聚焦三件事：项目创新点、视频制作规范流程、前后端工程创新写法。

## 1. 项目定位

- 目标用户：文旅内容运营、景区宣传、轻量视频创作者
- 核心问题：传统视频制作成本高、链路长、跨角色协作效率低
- 项目目标：用“对话 + 脚本 + 异步生成”把视频制作流程标准化、可复用

## 2. 项目创新点

### 2.1 产品创新

- 将复杂的视频生产流程拆成“信息采集 -> 对话澄清 -> 脚本生成 -> 参数配置 -> 视频任务 -> 结果输出”的可视化步骤。
- 把 AI 能力嵌入业务节点，而不是单点问答，降低用户表达门槛。
- 支持官方模式与个性化模式，兼顾标准内容生产和自由创作。

### 2.2 流程创新

- 把“创意输入”转成“结构化任务数据（task_data）”，在页面间持续复用。
- 视频生成采用异步任务机制，前端无阻塞体验，后端可独立扩展算力与调度策略。
- 用状态机思路管理任务状态：processing / complete / failed，便于监控和回溯。

### 2.3 工程创新

- 前端统一请求封装与统一任务上下文，减少页面间重复拼装参数。
- 后端与模型提供商解耦，支持替换模型平台而不改前端协议。
- 接口契约清晰，便于多人并行开发和联调。

## 3. 视频制作规范流程（标准化 S0-S6）

### S0 用户与素材准备

- 输入：昵称、头像、景点/主题基础信息
- 输出：用户画像与素材引用信息

### S1 创作意图澄清（对话）

- 输入：用户自然语言需求
- 处理：多轮对话补全拍摄风格、情绪、时长、叙事结构
- 输出：结构化创作需求 request

### S2 脚本生成与确认

- 输入：request + 场景信息
- 处理：生成脚本并允许编辑修订
- 输出：可执行脚本 scriptContent

### S3 视频参数配置

- 输入：脚本、风格、分辨率、时长、可选音频
- 输出：video_request（面向视频模型的参数集）

### S4 异步任务创建

- 前端调用 api/video，后端即时返回 video_status: processing 与 task_id
- 后端后台调用模型服务并持续更新状态

### S5 状态轮询与进度反馈

- 前端轮询 api/video/status
- processing：展示进度；complete：获得 video_url；failed：提示重试或降级方案

### S6 结果输出与复用

- 输出：视频地址、封面、可分享文案
- 可扩展：沉淀模板、风格标签、提示词策略，用于下一轮创作

## 4. 前端创新写法（当前项目实践）

- 统一任务上下文：以 app.globalData.task_data 贯穿多个页面，减少重复状态搬运。
- 请求抽象复用：对话页将初始化请求与用户对话请求复用同一发送函数，降低重复逻辑。
- 异步体验优化：等待页采用“真实状态轮询 + 进度展示”的方式减弱等待焦虑。
- 容错解析策略：对后端返回结构做兜底处理，兼容字符串与对象响应。
- 交互一致性：loading、bot typing、失败提示等 UI 状态统一控制，避免并发误触。

## 5. 后端创新写法（建议与对齐方向）

- 异步任务编排：创建任务接口只做接单与入队，立即返回 processing，避免请求阻塞。
- 供应商解耦：通过 provider adapter 映射不同模型平台返回格式，统一对外协议。
- 状态机与幂等：任务状态单向流转，重复提交可按 task_id 幂等处理。
- 轮询与回调兼容：优先后台轮询模型状态，可扩展回调加速状态收敛。
- 观测性建设：记录任务耗时、失败码、重试次数，支撑稳定性优化。

## 6. 接口契约（联调建议）

### 文案接口

- POST api/share
- 响应示例：

```json
{ "success": true, "response": "..." }
```

### 视频任务创建接口

- POST api/video
- 响应示例（即时返回）：

```json
{ "success": true, "video_status": "processing", "task_id": "..." }
```

### 视频状态接口

- GET api/video/status
- processing：

```json
{ "success": true, "video_status": "processing", "progress": 42 }
```

- complete：

```json
{ "success": true, "video_status": "complete", "video_url": "...", "progress": 100 }
```

- failed：

```json
{ "success": false, "video_status": "failed", "error_message": "..." }
```

## 7. 快速启动（真实预期）

- 首次搭建与联调一般需要 10-20 分钟（取决于后端服务可用性）。
- 基本步骤：安装依赖 -> 微信开发者工具导入 -> 构建 npm -> 配置后端域名 -> 编译运行。

## 8. 项目价值总结（答辩结尾可直接使用）

- 在业务层面：把视频创作从“经验驱动”转为“流程驱动”，提升稳定产出能力。
- 在工程层面：形成了可扩展的异步任务架构与清晰接口边界。
- 在落地层面：已具备继续演进为生产级系统的基础（调度、监控、治理）。
